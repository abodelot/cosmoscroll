How to play?

Game modes

CosmoScroll comes with two game modes:

  • Story Mode:
    Complete all levels and beat the final boss!
    Collect credits and use them for upgrading your spaceship and buying better weapons.
  • Arcade mode:
    Hold as long as possible against more and more powerful bad guys.
    Try to establish a new high score share it online to compete against other players!
    Don't forget to upgrade your spaceship in the Story Mode for improving your score.

Keyboard layout

Action Key
Move spaceship Directional arrows (up, down, left, right)
Main weapon Space
Fire a missile A
Active a cooler power-up Left Control
Pause the game Escap
Place HUD at the top of the screen Page Up
Place HUD at the bottom of the screen Page Down
Take a screenshot (saved in screenshots/) F1

The game can also be played with a pad or a joystick.
Keyboard and joystick layouts can be configured in the Options menu.

Power-ups

Power-ups are randomly dropped when an enemy is destroyed.

  • HP: restores 1 health point.
  • Shield: restores 1 shield point.
  • Cooler: gives +1 Cooler. Use a cooler to instantly cancel over-heat. You can store up to 3 coolers.
  • Missile: gives +1 Missile. A deadly weapon with a large damage zone. You can store up to 3 missiles.
  • Double shot: doubles your weapons for 10 seconds (cannot be combined with triple shot)
  • Triple shot: triples your weapons for 10 seconds (cannot be combined with double shot).
  • Speed: increases your speed for 10 seconds
  • Super Banana: restores all health points and all shield points

Where is the configuration file?

CosmoScroll saves your configuration in the settings directory, according to your operating system:

  • Windows : Application Data/cosmoscroll/cosmoscroll.ini
  • Linux : ~/.config/cosmoscroll/cosmoscroll.ini
  • Mac OS : ~/Library/Application Support/cosmoscroll/cosmoscroll.ini

If you don't want ComsoScroll storing your configuration in this directory, you can set another location to save this file, using -config option in command line (see command line arguments section for more details).

If you want to reset the game, simply delete this file, a new one will be created next time you launch the game.
Take note you will lost your entire configuration (scores, level progression, settings, etc.) by deleting this file.

Command line arguments

Option Role
-config <path> Override configuration file default location.
If path is a directory, the game will look for a configuration file named cosmoscroll.ini (or create it if it doesn't exist).
If it is a regular file, it will use it as an alternate configuration file.
-res <path> Override resources directory default location.
Use this option if you want to install CosmoScroll exectuable and the "resources" directory in different locations.
-version Show CosmoScroll version
-help Show usage

I've found a bug! / I've a feature suggestion!

If you have encountered a problem while playing CosmoScroll, please report it on the bug tracker (if the issue has not been reported yet).
If possible, please clearly mention the steps to reproduce the bug.

You can also contact the main developer of CosmoScroll (alexandre.bodelot@gmail.com) if you want to submit a feature suggestion, it will be welcomed!

If CosmoScroll isn't available in your native language, you can translate it by providing the language file (see resources/lang/ directory) and send it to the CosmoScroll team for inclusion in further releases.

How to compile the game from source?

CosmoScroll compiles on GNU/Linux (GCC), Mac OS (GCC) and Windows (mingw32, Visual Studio 2008).
Other systems and compilers haven't been tested.

First, you must install the following libraries:

  • SFML, version 2.0
  • libdumb (for Debian/Ubuntu, simply install libdumb1-dev package)

The easiest way to get the source code is to checkout the project's git repository (follow instructions on this page).

You can compile the game with Code::Blocks IDE as a project file is already provided (cosmoscroll.cbp).
The makefile generator (make_generator.py, written in Python) can also produce UNIX makefiles (work with both Linux and Mac OS).

Comment jouer ?

Modes de jeu

Deux modes de jeu vous sont proposés :

  • Mode histoire :
    Terminez tous les niveaux jusqu'à l'affrontement avec le boss final !
    Collectez des crédits pour améliorer votre vaisseau et acheter de meilleures armes.
  • Mode arcade :
    Tenez le plus longtemps possible face à des ennemis de plus en plus nombreux et de plus en plus puissants.
    Tentez d'établir un nouveau record et envoyer le en ligne afin de vous mesurer aux autres joueurs !
    N'oubliez pas d'améliorer votre vaisseau dans le Mode Histoire pour avoir un meilleur score.

Commandes clavier

Action Touche
Déplacer le vaisseau Flèches directionnelles (haut, bas, gauche, droite)
Tir principal Espace
Tirer un missile A
Utiliser bonus Glaçon Contrôle gauche
Mettre le jeu en pause Échap
Positionner le panneau de contrôle en haut de l'écran Page Haut
Positionner le panneau de contrôle en bas de l'écran Page Bas
Prendre une capture d'écran (enregistrée dans screenshots/) F1

Le jeu peut également être contrôlé avec un joystick ou une manette de jeu.
Les touches du clavier et les boutons du contrôleur peuvent être redéfinis dans le menu Options.

Bonus

  • Point de vie : restaure 1 point de vie du vaisseau.
  • Bouclier : restaure 1 point de bouclier.
  • Glaçon : donne +1 Glaçon. Utiliser un glaçon permet d'annuler les effets de la surchauffe. Jusqu'à 3 glaçons peuvent être cumulés.
  • Missile : donne +1 Missile. Arme très puissante avec une large zone de dégâts. Jusqu'à 3 missiles peuvent être cumulés.
  • Double tir : double vos tirs pendant 10 secondes.
  • Triple tir : triple vos tirs pendant 10 secondes.
  • Vitesse : multiplie par 2 votre vitesse pendant 10 secondes
  • Super Banane : restaure tous les points de vie et points de bouclier

Où se trouve le fichier de configuration ?

CosmoScroll enregistre votre configuration dans le répertoire des préférences de votre système d'exploitation :

  • Windows : Application Data/cosmoscroll/cosmoscroll.ini
  • Linux : ~/.config/cosmoscroll/cosmoscroll.ini
  • Mac OS : ~/Library/Application Support/cosmoscroll/cosmoscroll.ini

Si vous ne voulez pas que ComsoScroll enregistre votre configuration dans ce dossier, vous pouvez indiquer un autre endroit pour enregistrer ce fichier avec l'option -config en ligne de commande (voir la section paramètres de la ligne de commande pour plus de détails).

Si vous voulez retrouver les paramètres par défaut de l'application, effacez ce fichier, un nouveau sera créé automatiquement à la prochaine utilisation du jeu.
Attention, en effaçant ce fichier, votre configuration (scores, progression dans les niveaux, préférences, etc.) sera perdue.

Paramètres de la ligne de commande

Option Rôle
-config <chemin> Indiquer l'emplacement du fichier de configuration.
Si chemin est un répertoire, le jeu va chercher un fichier un fichier de configuration nommé cosmoscroll.ini (ou le créer s'il n'existe pas).
Si c'est un fichier, le jeu l'utilisera comme fichier de configuration alternatif.
-res <chemin> Indiquer l'emplacement du répertoire des ressources.
Utiliser cette option si vous voulez installer l'exécutable de CosmoScroll et le répertoire "resources" dans différents endroits.
-version Affiche la version de CosmoScroll
-help Affiche l'usage

Comment compiler le jeu depuis les sources ?

CosmoScroll compile sous les plateformes GNU/Linux (GCC), Mac OS (GCC) et Windows (mingw32, Visual Studio 2008).
Les autres systèmes et compilateurs n'ont pas été testés.

Vous devez tout d'abord installer les bibliothèques suivantes :

  • SFML, version 2.0
  • libdumb (pour Debian/Ubuntu, installer le paquet libdumb1-dev)

Le moyen le plus simple de récupérer les sources est de faire un checkout sur le dépôt git du projet (reportez-vous aux instructions de la page).

Un fichier projet Code::Blocks (cosmoscroll.cbp) est fourni et permet de compiler le jeu depuis cet IDE.
Un générateur de makefile en Python est également fourni avec les sources du projet (make_generator.py)
Le makefile généré permet de compiler le jeu sous Linux et Mac OS.
